# Объяснение: Почему в Pull Request видны не все файлы

## Как работают Pull Requests

Pull Request показывает **только изменения (diff)** между двумя ветками:
- **base branch** (обычно `main`) - базовая ветка
- **compare branch** (ваша ветка, например `feature/improvements`) - ветка с изменениями

### Это нормальное поведение!

Если в PR видны только 4 файла, это означает, что:
- Эти 4 файла были изменены или добавлены в ветке `feature/improvements`
- Остальные файлы идентичны в обеих ветках
- GitHub показывает только различия, чтобы не загромождать интерфейс

## Как увидеть все файлы проекта

### Вариант 1: Просмотр репозитория (не через PR)
1. Перейдите на https://github.com/doboleshev/diary-app
2. Нажмите на вкладку **"Code"** (не "Pull requests")
3. Там вы увидите все файлы проекта

### Вариант 2: Просмотр конкретной ветки
1. На странице репозитория нажмите на выпадающий список веток
2. Выберите ветку `feature/improvements`
3. Вы увидите все файлы в этой ветке

### Вариант 3: Если нужно, чтобы все файлы были в PR

Если вы хотите, чтобы в PR были видны все файлы проекта, это означает, что в ветке `main` их нет. В этом случае:

```bash
# 1. Убедитесь, что вы на feature/improvements
git checkout feature/improvements

# 2. Добавьте все файлы
git add .

# 3. Создайте коммит
git commit -m "Add all project files"

# 4. Загрузите на GitHub
git push origin feature/improvements
```

## Текущая ситуация

Если в PR видны только 4 файла, это означает:
- ✅ Проект уже есть в ветке `main`
- ✅ В ветке `feature/improvements` изменены только 4 файла
- ✅ Это правильное поведение для PR

## Что показывается в "Files changed"

Вкладка "Files changed" в Pull Request показывает:
- **Зеленые файлы** - новые файлы, добавленные в feature ветке
- **Желтые файлы** - измененные файлы
- **Красные файлы** - удаленные файлы

Файлы, которые не изменились, не показываются - это нормально!

## Если нужно увидеть все файлы в PR

Если вы действительно хотите, чтобы все файлы были видны в PR (например, для ревью всего проекта), можно:

1. **Создать новый PR с полным проектом:**
   ```bash
   # Создайте новую ветку
   git checkout -b feature/full-project
   
   # Убедитесь, что все файлы добавлены
   git add .
   git commit -m "Add complete project"
   git push -u origin feature/full-project
   ```

2. **Или добавить все файлы в текущую ветку:**
   ```bash
   git checkout feature/improvements
   git add .
   git commit -m "Include all project files in PR"
   git push origin feature/improvements
   ```

Но обычно это не нужно - PR должен показывать только изменения!
